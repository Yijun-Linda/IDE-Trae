## 一、核心思路（已确认）

**从结果导向 → 过程导向**

将"1x 拿铁（去冰，燕麦奶）"翻译为"拿冰杯→加冰块→加燕麦奶→倒浓缩"

**关键洞察**：
- 不是增加新设备，而是改造已有工具（小票）
- 纸质小票具有物理实体感，可以拿着、打勾
- 符合杨老师"不要搞特殊"的原则

## 二、方案架构（已实现）

```
[外卖平台Webhook] → [AI认知翻译引擎] → [云打印机API] → [动作脚本小票]
                          ↓
                   [通义千问/豆包API]
```

**核心模块**：
- `webhook/handler.py`：接收外卖平台订单
- `translator/translator.py`：AI订单拆解
- `printer/yilianyun.py` + `printer/feie.py`：云打印机对接
- `queue/manager.py`：队列管理（时间延迟出单）

## 三、用户画像（已校准）

### 目标人群（已明确）

**轻度智力障碍者**：
- 能正常对话、正常沟通
- 多线程任务会手忙脚乱
- 旺季高负荷时认知超载

**不适用人群**：
- 中度心智障碍（需要更多支持）
- 听力障碍（依赖视觉）
- 主动检索能力弱的人群

### 关键决策者

**杨老师（辅导员）**：
- 需要灵活配置系统
- 负责处理异常情况
- 评估员工使用效果

## 五、待解决问题（已识别）

### 问题 1：遗忘-主动

**现象**：如果员工根本不记得去看小票，所有提示失效

**杨老师的实践**：
- 墙上贴纸条：失败
- 拍视频给员工看：效果有限
- 视频检查机制：有效但需要辅导员在场

**可能的答案**：
- 轻度智力障碍者可能比中重度更容易建立看小票的习惯
- 旺季高负荷时，小票的存在本身就在"提醒他们看"
- 第三次忘记时给予更强的干预

**待验证**：这个假设是否成立？

---

### 问题 2：按需触发的设计

**现象**：当认知超载时，人很难自主判断"我需要帮助"

**杨老师的场景**：
> 中午她出去买个饭，刚走到半路，来自轻度心智障碍的员工的电话就打过来了

**这说明**：他们在慌乱时，目前会自主求助辅导老师

**可能的答案**：
- 被动推送解决"我忘了看"的问题
- 但无法解决"我根本不知道我需要看"的问题
- 需要外部触发（如队列延迟出单）

**待探索**：是否有更好的"按需触发"机制？

---

### 问题 3：用人单位动机

**现象**：普通咖啡店为什么要雇用轻度智力障碍员工？

**已识别的政策杠杆**：
- 残障就业保障金减免
- 增值税优惠
- CSR叙事价值（品牌形象）

**关键问题**：
- 轻度智力障碍在残障评定体系中属于几级？享受同等政策吗？
- 补贴额度是否足以覆盖"管理成本"溢价？
- 政策覆盖的岗位类型是什么？

**待探索**：政策细节 + 用人单位实际动机

## 六、与摄像头方案的关系

云打印机方案是**基础方案**，摄像头方案是**进阶方案**：

| 维度 | 云打印机 | 摄像头+语音 |
|------|---------|-----------|
| **触发** | 被动（订单） | 主动（行为检测） |
| **主动性要求** | 员工主动看 | 系统主动介入 |
| **技术复杂度** | 低 | 高 |
| **部署成本** | 低 | 高 |
| **可解释性** | 高 | 中 |

**可能的组合**：
- 平时用云打印机（低成本、低侵入）
- 摄像头检测异常（主动感知）
- 异常时语音介入或云打印机打印检查清单

## 七、长期目标（已明确）

**"优化的目的是没有优化"** —— 杨老师

**最终目标**：
- 不是让工具成为永久依赖
- 而是让他们在需要时看一眼/听一下
- 最终内化能力，不再需要工具

**成功定义**：
- 辅助的使用频率逐渐降低
- 而不是辅助本身的覆盖率提升

## 八、比赛叙事

**核心叙事**：
> "AI不是替代心智障碍者的工作，而是翻译世界给他们能懂的语言。"

**技术赋能人力，而非替代人力**

**关键对比**：
- 日本：用机器人替代人力
- 你的方案：用AI赋能人力，让他们能做普通人能做的事

**挑战**：
- 如果工具让他们"和普通人一样工作"，为什么还需要他们？
- 答案：残障补贴 + CSR价值 + 政策指标
