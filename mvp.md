# AI认知翻译官：心智障碍咖啡店订单处理系统

## 项目信息

- **项目名称**：AI认知翻译官（AI Cognitive Translator）
- **目标用户**：上海塘桥街道慈善咖啡店心智障碍员工及辅导员杨老师
- **核心场景**：将外卖/线下订单翻译为心智障碍员工可执行的动作脚本
- **生成日期**：2026-04-23
- **版本**：v0.1（比赛MVP）

***

## 一、产品定义

### 1.1 核心问题

心智障碍员工在咖啡店工作时面临的核心障碍：

- **结果→过程的认知转换困难**：普通小票显示"1x 拿铁（去冰，燕麦奶）"，员工需要自己拆解成动作步骤
- **多订单并行导致认知超载**：当多个订单同时涌入，工作记忆通道被堵塞，容易搞混、重复做

### 1.2 解决方案

AI部署在收银系统和云打印机之间，作为认知翻译层：

- **输入**：外卖平台/线下收银系统的原始订单数据
- **处理**：AI将结果导向的订单翻译为过程导向的动作脚本
- **输出**：通过云打印机API输出重构后的小票

**关键创新**：不是给员工增加新设备，而是改造他们已经在使用的工具（小票），把世界翻译成他们能懂的语言。

### 1.3 产品形态（比赛MVP）

**硬件层**：
- 云打印机（已存在，对接美团/饿了么等平台）

**软件层**：
- AI认知翻译引擎（Webhook接收订单 → Prompt工程拆解步骤 → 调用打印机API）
- 打印队列管理（时间延迟自动出单：每杯制作完成后5分钟自动出下一单）
- 异常回退（AI服务异常时自动打印原始小票）

**交互层**：
- 员工：看小票、做咖啡
- 杨老师：处理异常、人工支持

## 二、交付物

### 2.1 核心交付物

**1. AI认知翻译引擎**
- 接收外卖平台Webhook
- 调用大模型API（通义千问/豆包）进行订单拆解
- 支持10个SKU（5个热饮+5个冷饮）
- 调用云打印机API输出动作脚本小票

**2. 动作脚本小票模板**
```
订单 #{order_id} | 第 {current}/{total} 杯

□ {步骤1}
□ {步骤2}
□ {步骤3}

5分钟后自动出下一杯
```

**3. 对比展示材料**
- 普通小票 vs AI翻译后小票的对比图

### 2.2 辅助交付物

**异常回退机制**
- AI服务异常时自动切换回原始小票模式
- 无日志记录（比赛阶段简化）

## 三、成功标准

### 3.1 用户层面

**心智障碍员工**：
- [ ] 能独立完成单杯饮品的制作（无需杨老师口头指导）
- [ ] 多订单场景下搞混、重复做的次数减少

**杨老师（辅导员）**：
- [ ] 同步打断式支持减少50%以上（通过主观反馈验证）
- [ ] 异常处理回归到人工支持（原料缺货、设备故障等由杨老师处理）

### 3.2 系统层面

- [ ] 订单翻译准确率 > 80%（10个SKU的步骤拆解正确）
- [ ] 打印延迟 < 5秒（从接单到出票）
- [ ] 系统可用性 > 95%（比赛演示期间）

### 3.3 社会价值层面

- [ ] 不增加员工的社会特殊感（外观与普通咖啡店无区别）
- [ ] 实施成本可控（慈善咖啡店可承受）

## 四、设计原则

1. **工作记忆是稀缺资源**
   - 一次只处理一杯，避免认知超载
   - 动作脚本不超过3步

2. **被动接收优于主动检索**
   - 小票自动打印，无需员工主动获取信息
   - 时间延迟自动出单，无需按钮操作

3. **不要搞特殊**
   - 不增加可见的新设备
   - 改造已有工具而非插入新工具

4. **结构同构性思维**
   - 将复杂订单结构同构为简单动作序列
   - 保留原始信息的同时降低认知维度

## 五、技术栈

- **后端**：Python/Node.js（Webhook接收、API调用）
- **AI模型**：通义千问/豆包（订单拆解Prompt工程）
- **打印机API**：主流云打印机厂商API（易联云、飞鹅等）
- **部署**：阿里云/腾讯云函数（低成本、免运维）

## 六、后续演进方向（v1.0及以后）

### v1.0 生产版本

**新增硬件**：
- 无线按钮（绿色=完成）：员工完成一杯后按一下，立即出下一单，替代时间延迟

**优化**：
- 杨老师配置界面（步骤模板配置）
- 规则引擎（减少大模型API调用成本）
- 异常日志记录

### v2.0 自动化版本

**自动化点一：订单翻译**
- Webhook接收订单 → AI翻译 → 调用打印机API
- 无需人工干预

**自动化点二：异常检测**
- 监控AI服务健康状态
- 异常时自动切换回原始小票模式
- 发送通知给杨老师

**自动化点三：数据收集**
- 自动记录每杯制作时间
- 自动统计错误率
- 生成周报给杨老师
