# MVP 规划：辅导员的 AI 数字化身

**规划来源**: Gemini + 用户协作  
**规划日期**: 2026-04-18  
**核心定位**: 辅导员的 AI 数字化身 —— 实时多模态容错系统

**技术栈更新**: 基于核心约束（10天开发周期、无多模态经验、TRAE SOLO公益赛道），技术方案调整为**阿里云视觉智能开放平台 + 普通摄像头**，替代原方案中的GPT-4V/Gemini等多模态VLM。

---

## 技术栈总览

| 组件 | 选型 | 原因 |
|------|------|------|

## 🚨 第零步：生死攸关的"感官与硬件可用性"测试（开发前必须做）

**不要立刻开始写代码！**

心智障碍群体（尤其是自闭症谱系）往往伴随**严重的感官敏感（Sensory Overload）**。

### 痛点假设
AR 眼镜的重量、镜片上的闪烁光影、耳边的突发电子音，可能会直接导致他们情绪崩溃或抗拒佩戴。

### MVP 验证动作（去现场的第一件事）
借一台 Rokid 眼镜（或者哪怕是一个普通的墨镜+蓝牙耳机），让青年试戴 15 分钟。观察他们是否抗拒。

### Plan B（防死预案）
如果他们抗拒戴眼镜，立刻将硬件方案降级为：**吧台正上方悬挂的广角摄像头 + 青年面前的 iPad/带屏音箱**。这同样能实现视觉识别，且完全不侵入青年的身体边界。

**技术实现**：使用阿里云视觉智能开放平台的人脸表情识别、人体行为分析API，通过普通摄像头捕获画面，实时分析青年状态。

## 🏃 第一阶段：核心功能一「AI 任务一键拆解 Agent」（3-5天可跑通）

这是你的**速赢点（Quick Win）**，技术门槛最低，但对辅导员的减负效果最明显。

### 产品形态
辅导员手机端的一个小程序/H5/Agent 对话框。

### 核心工作流

**1. Input (语音输入)**
辅导员说："今天给王先生做一杯热拿铁，要低因豆，记得最后拉个心形。"

**2. Process (LLM 信息抽取与编排)**
使用阿里云通义千问或豆包大模型，配合明确的 Prompt。

*Prompt 示例*：
```
你是一个特殊教育任务拆解专家。请将以下语音指令拆解为适合智力障碍青年的 4-6 个原子步骤。过滤掉无关信息（如'今天'、'王先生'），高亮关键动作和物品。输出 JSON 格式。
```

**技术实现**：调用阿里云百炼平台或直接使用通义千问API，中文场景优化，响应速度快。

**3. Output (多模态生成)**
将 JSON 步骤映射到预设的图片库（例如：[低因豆盒子图片] -> [咖啡机按键图片] -> [拉花缸图片]）。如果是高阶配置，可以调用文生图 API 快速生成缺失动作。

### MVP 交付标准
辅导员说完话后，3 秒内，青年的设备端（眼镜或 iPad）出现极其清晰的步骤卡片，青年可通过语音说"下一步"或点头（Rokid 陀螺仪）翻页。

## 🧗 第二阶段：核心功能二「AI 视觉动作容错系统」（比赛的核心壁垒）

这是评委最看重的"硬核技术"部分。但如果你试图做"24 小时无延迟连续视频流动作识别"，你会死于延迟和算力成本。**MVP 需要"讨巧"**。

### 产品形态
基于步骤的**关键帧状态校验（State Validation）**，而不是连续动作跟踪。

### 核心工作流（破局思路）

**1. 不要做连续识别**
假设青年正在做"打奶泡"这一步。不要让 VLM 每秒看 30 帧视频（延迟太高）。

**2. 触发式截帧（Trigger-based）**
当青年在当前步骤停留超过 30 秒（超时卡壳），或者青年主动说"我看下/对不对"时，Rokid 摄像头截取当前视角的**高清图片（关键帧）**。

**3. 视觉智能分析**
将该图片发送给阿里云视觉智能开放平台，调用**人体行为分析**和**人脸表情识别**API。

**技术实现**：
- **人体行为分析**：检测青年当前动作（站立/坐/手部动作），判断是否完成当前步骤
- **人脸表情识别**：检测困惑、焦虑等情绪信号，判断是否需要干预
- **注意力检测**：判断青年是否专注于当前任务

**判断逻辑示例**：
```
if (当前步骤 == "将牛奶倒入拉花缸") {
  if (行为分析.检测到手中物体 != "牛奶盒") {
    触发纠正提示("停，牛奶拿错了，拿绿色的盒子");
  }
  if (表情识别.置信度 > 0.7 && 表情 == "困惑") {
    触发帮助提示("需要我重复一遍吗？");
  }
}
```

**4. 干预反馈**
通过**阿里云语音合成API**播放语音："停，牛奶拿错了，拿绿色的盒子。"并在iPad/眼镜视野中显示对应步骤的提示图片。

**技术实现**：
- **语音合成**：调用阿里云语音合成API，将文本转为自然语音
- **视觉提示**：在iPad屏幕上显示当前步骤的参考图片，用红框标注正确物品
- **延迟目标**：整体响应时间 ≤ 3秒（视觉识别1秒 + 语音合成1秒 + 网络传输1秒）

### MVP 交付标准
在比赛现场/视频演示中，针对**"做一杯美式咖啡"这一个固定场景**，演示 2 次经典的易犯错误（例如：冰块加多了、按错了萃取键），系统能够准确抓拍并在 5 秒内给出语音纠正。

## 🏆 第三阶段：比赛路演（Pitch）与故事线设计

你的项目不能只是一堆代码，它必须是一个能打动人心的"闭环"。在提交作品时，请严格按照以下叙事结构：

### 1. 痛点反转
开场不要讲"心智障碍者多可怜"，而是讲"辅导员多崩溃"。展示一张图：1 个辅导员被 5 个出错的青年包围，焦头烂额。

### 2. 方案亮相
引出"多模态 AI 辅导员化身"。强调我们不是要"替代"辅导员，而是**"复制辅导员的注意力和耐心"**。

### 3. 核心 Demo（高能时刻）

**Demo A（效率提升）**：现场演示一句话拆解复杂订单，直接推送到终端。

**Demo B（兜底容错）**：演示视角切换至 Rokid 第一人称。演示者故意拿错糖浆，屏幕闪烁并传来温和的语音提醒："等一下，拿左边那个红色的瓶子哦。"

### 4. 成本与社会价值
强调利用**阿里云视觉智能API + 现成硬件**，将原本几十万的特殊教育系统成本，打下来变成每月几十块钱的API调用费。这才是"科技致善"。

**成本对比**：
| 方案 | 硬件成本 | 软件成本 | 部署周期 |
|------|---------|---------|---------|
| 传统定制系统 | 10万+ | 20万+ | 3-6个月 |
| 本方案（阿里云API） | 普通摄像头/iPad | 按调用量付费 | 10天 |

## 🛠️ 行动清单（接下来的 72 小时）

### 1. 硬件借测
立刻搞定测试设备（如果是 AR 眼镜，去确认它的 SDK 能否方便地调用摄像头和麦克风；如果搞不定，立刻切换到 iPad 方案，不影响你拿奖）。

### 2. 工作流标定
去梦工坊，死盯着一个最常出错的青年，记录他做某一款咖啡的**完整 10 个步骤**，以及他最容易在哪 2 个步骤出错。这就是你 MVP 里的唯一测试用例。

### 3. 跑通阿里云视觉智能 API
注册阿里云账号，开通视觉智能开放平台，测试以下核心能力：

**测试清单**：
1. **人脸表情识别**：上传一张"困惑表情"的照片，测试能否识别出"困惑"标签
2. **人体行为分析**：上传一张"手持牛奶盒"的照片，测试能否识别出"手持物体"行为
3. **语音合成**：测试将"停，牛奶拿错了"转为语音，检查自然度和延迟
4. **免费额度评估**：确认参赛期间（到5月10日）的免费调用额度是否充足

**验证标准**：如果阿里云API能准确识别出测试图片中的关键信息，你的核心技术逻辑就成立了。
