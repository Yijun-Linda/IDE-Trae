## 项目背景

本项目是 **TRAE SOLO 挑战赛「Hello AI 科技致善」公益赛道** 参赛作品，聚焦**心智障碍群体就业支持**命题。

### 核心问题

心智障碍青年（包括唐氏综合征、自闭症等）在就业场景中面临巨大挑战。据实地观察，能在慈善商店正常就业的唐氏综合征或自闭症青年，可能不到该群体的 1%。

### 研究方向演进

**初始方向**：探索如何利用 AI 技术（计算机视觉、语音交互、大语言模型）为心智障碍青年提供实时任务支持

**当前方向**：基于实地采访的深刻洞察，产品定位从"技术增强"转向"隐形辅助"——不是提供一种更好的辅助工具，而是让辅助工具的存在逐渐变得不必要

### 核心洞察

**社会化的本质是不要搞特殊**（来自杨老师，慈善咖啡店店长）

真正的包容不是创造一个特殊的环境，而是把心智障碍者放在真实的世界里，相信他们能够适应。这改变了产品的根本设计哲学：辅助工具必须足够隐形、足够通用，成为工作环境的自然组成部分，而不是让员工显得特殊的标记。

## 产品定义

### 核心功能

一个基于微信小程序的轻量工具，核心交互是**扫码获取即时步骤引导**。

**为什么是小程序**：

- 不需要下载安装，扫码即用
- 开发周期短，符合18天比赛周期
- 可以嵌入语音播放
- 界面普通，不会让员工显得特殊

### 设计原则

1. **触发方式触手可见**：贴在设备旁的二维码，员工不需要记得什么时候用，看到不确定就扫
2. **输出简短流程化**：不是解释原理，而是明确的第一步、第二步、第三步
3. **内容与真实环境一致**：基于具体咖啡店的设备型号、摆放位置、流程定制
4. **辅导员可更新**：杨老师能在5分钟内把新问题转化为引导步骤

### 成功标准

不是心智障碍员工使用了多少次辅助工具，而是他们使用辅助工具的频率是否在逐渐降低。**辅助的可消退性比辅助本身更重要**。

## 技术栈

| 组件       | 选型               | 原因             |
| -------- | ---------------- | -------------- |
| **前端**   | 微信小程序            | 无需安装、扫码即用、开发快速 |
| **语音播放** | 微信小程序内置音频        | 无需额外SDK        |
| **内容管理** | 轻量CMS或Markdown文件 | 杨老师可快速更新       |
| **开发工具** | Trae SOLO        | 比赛要求           |

**技术复杂度降级说明**：

技术方案从"阿里云视觉智能 + 实时多模态识别"降级为"微信小程序扫码 + 语音播放"。原因：

1. 采访发现心智障碍员工对"触手可见的提醒"响应最好，复杂技术反而增加认知负担
2. 18天周期内，简单方案更有可能完成并验证
3. 产品成功不依赖技术复杂度，而依赖内容质量和用户适配

## 核心洞察摘要

### 来自杨老师的五个真实需求

1. **流程拆解与步骤化引导**：咖啡师岗位需要同时处理多维度信息（冷热、冰量、杯型、堂食/外带、制作顺序）
2. **多任务排序与优先级管理**：多个客人同时点单时容易花，需要明确的顺序指引
3. **环境状态识别与检查清单**：关机问题反复出现，需要可感知的检查点而非抽象概念
4. **沟通辅助与需求表达**：引导客人做选择的话术模板
5. **辅导员远程支持与异步协作**：减少同步打断式支持

### 设计约束

- 教他们一定要简短、流程化、短语式，最好是动作指令
- 只要不是触手可见的提醒，他们都很难主动去查阅
- 职业康复不等于就业，零支持的百分百脱手几乎不可能
- 优化的目的是没有优化
