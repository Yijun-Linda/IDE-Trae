## 二、比赛提交要求分析

### 2.1 提交格式（来自trae-solo-challenge-hello-ai.md）

**发帖标题格式**：
```
【Hello AI 科技致善】+ 一句话亮点
```

**正文内容结构**：
1. **摘要**（2句话：服务对象+解决什么问题+带来什么效果）
2. **真实场景与需求**（目标人群、痛点描述、现有做法及不足）
3. **作品介绍**（产品形态、核心功能、技术实现简述）
4. **使用演示**（截图/录屏/流程图）
5. **收获与反思**（最大挑战、下一步优化）

### 2.2 评选标准

- **切题**：是否精准回应心智障碍群体就业支持的真实需求
- **实战**：是否有真实使用场景，是否经过实际验证
- **过程**：开发过程是否清晰，问题与解决思路是否可追溯
- **社会价值**：能否切实帮助目标人群，解决真实痛点

### 2.3 提高入围概率的关键

**内容层面**：
- 痛点要具体（不是泛泛而谈"帮助弱势群体"）
- 方案要落地（展示真实使用场景，有截图/录屏/用户反馈）
- 过程要完整（从发现问题到解决问题的完整思考链条）

**形式层面**：
- 标题要有吸引力（一句话说清核心价值）
- 结构要清晰（用小标题、分段）
- 配图要到位（一图胜千言）

**加分项**：
- 有真实用户测试或反馈
- 有数据支撑效果
- 有明确的后续迭代计划

### 3.2 详细执行计划

#### 阶段1：代码验证与修复（2-3小时）

**目标**：确保核心流程可以跑通

**任务清单**：

**成功标准**：

#### 阶段2：展示材料制作（2-3小时）

**目标**：制作比赛提交所需的核心视觉材料

**任务清单**：
- [ ] 制作普通小票 vs AI翻译小票的对比图
- [ ] 设计系统架构流程图
- [ ] 准备实地调研照片（如有）
- [ ] 制作Before/After场景对比图

**设计规格**：
- 对比海报：1200 x 630px（社交媒体分享图）
- 配色：温暖米色背景，咖啡棕主色调
- 布局：左侧"混乱"（普通小票堆叠），右侧"清晰"（AI翻译后小票）

#### 阶段3：比赛提交帖子撰写（2-3小时）

**目标**：按照官方格式撰写完整的提交帖子

**内容结构**：

**① 摘要**
```
面向上海塘桥街道慈善咖啡店的心智障碍员工，在旺季多订单场景下提供AI订单拆解和分步指引，
目前通过实地调研和系统原型验证，可将复杂订单翻译为每步不超过10个字的动作脚本。
```

**② 真实场景与需求**
- 目标人群：轻度心智障碍咖啡师（能正常对话，但多线程任务会手忙脚乱）
- 痛点描述：旺季多订单涌入时，工作记忆通道被堵塞，容易搞混、重复做
- 现有做法：辅导员杨老师口头指导，但同步打断式支持成本高

**③ 作品介绍**
- 产品形态：部署在收银系统和云打印机之间的AI翻译层
- 核心功能：将"1x 拿铁（去冰，燕麦奶）"翻译为"拿冰杯→加冰块→加燕麦奶→倒浓缩"
- 技术实现：Python + 通义千问API + 云打印机API，时间延迟自动出单

**④ 使用演示**
- 截图：普通小票 vs AI翻译小票对比
- 截图：系统架构图
- 截图：实地调研照片

**⑤ 收获与反思**
- 最大挑战：如何在"不要搞特殊"的原则下设计辅助工具
- 解决方案：改造已有工具（小票）而非增加新设备
- 下一步：实地测试验证效果，根据反馈优化Prompt

#### 阶段4：README.md完善（1小时）

**目标**：让项目主页清晰展示项目价值

**内容要点**：
- 项目一句话介绍
- 核心问题与解决方案
- 快速开始指南
- 项目结构说明
- 相关文档链接

#### 阶段5：最终检查与提交（1小时）

**提交前检查清单**：
- [ ] 帖子内容完整，符合格式要求
- [ ] 配图清晰，尺寸正确
- [ ] 代码可以运行（至少核心模块）
- [ ] README.md已更新
- [ ] 所有相关文档已链接
